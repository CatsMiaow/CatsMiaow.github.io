# 관리

## 내서버

서버의 봇 설정을 보여주거나 알림을 받을 관리자를 등록합니다.

- !내서버
  - 봇 기능에서 관리자가 설정한 항목을 보여줍니다.
  - 등록한 관리자는 `바지사장`으로 표시됩니다.
- !내서버 관리자 `@멘션`
  - 멘션(@로 호출)으로 관리자를 등록합니다, 디스코드 관리자와 무관합니다.
  - 등록된 관리자는 `!공지`, `!역할` 기능 등에서 문제가 발생했을 때 알림을 받아볼 수 있습니다.
- !내서버 관리자 삭제
  - 등록된 관리자를 삭제합니다.
- !내서버 기록 `페이지:숫자:옵션` `@멘션:옵션`
  - 현재 서버에서 입력된 명령어 사용 내역을 보여줍니다.
  - @멘션으로 특정 유저만 필터링하여 관리 용도로 사용할 수 있습니다.
  - 목록 내의 명령어 링크를 클릭하여 해당 메시지로 이동할 수 있습니다.
  - 내역은 일정 기간 보관하고 삭제됩니다.

> !내서버 \
> !내서버 관리자 @테스트#1234 \
> !내서버 관리자 삭제 \
> !내서버 기록 \
> !내서버 기록 @테스트#1234

## 공지

서버에 뉴비(새내기)가 입장할 때 보여줄 메시지를 등록합니다.

- !공지
  - 등록된 공지를 보여줍니다.
- !공지 `내용`
  - 공지 내용을 등록합니다. 여러 줄 입력은 `Shift + Enter`로 구분합니다.
  - 공지를 등록한 채널에서 메시지를 보여줍니다.
- !공지 삭제
  - 등록된 공지를 삭제합니다. 메시지를 보여주지 않습니다.

> !공지 \
> !공지 테스트입니다. \
> !공지 삭제

## 역할

### 기본 역할

서버에 뉴비(새내기)가 입장할 때 지정한 역할을 부여합니다.

- !역할 기본 `역할명`
  - 봇이 `역할 관리` 권한을 가지고 있어야 하고, 봇이 가진 역할보다 낮은 역할만 설정할 수 있습니다.
  - 기본 역할을 `@everyone`이 아닌 다른 역할을 부여하여 역할 이름/색상/설정을 변경할 수 있습니다.
- !역할 기본 삭제
  - 지정된 기본 역할을 삭제합니다.

### 부여 역할

다른 유저에게 부여할 수 있는 역할을 설정합니다.

- !역할 부여 `역할명`
  - 봇보다 낮은 역할만 설정할 수 있습니다.
  - 특정 역할을 지정하여 특정 유저가 다른 유저에게 수동으로 역할을 부여할 때 사용할 수 있습니다.
- !역할 부여 `@멘션`
  - 역할 부여를 설정한 후에 사용할 수 있습니다.
  - 봇보다 높은 역할의 유저만 사용할 수 있습니다.
- !역할 부여 삭제
  - 설정된 역할 부여를 삭제합니다.

### 요청 역할

유저가 스스로 역할을 가져갈 수 있도록 설정합니다.

- !역할 요청 교체
  - 봇이 가진 역할보다 낮은 역할만 가져갈 수 있습니다.
  - 기존 역할을 요청하는 역할로 교체합니다.
  - 역할을 교체하기 때문에 기존 역할이 봇보다 높으면 수행할 수 없습니다.
- !역할 요청 추가
  - 역할을 교체하지 않고 기존 역할에 역할을 추가하도록 설정합니다.
  - 나머지는 위의 교체 설정과 같습니다.
- !역할 요청 삭제
  - 설정된 역할 요청을 삭제합니다.
- !역할 요청 `역할명`
  - 역할 요청을 설정한 후에 사용할 수 있습니다.
  - 서버에 처음 입장한 유저가 진영(?)을 선택할 때 사용할 수 있습니다.
- !역할 삭제 `역할명`
  - 요청한 역할을 삭제할 수 있습니다.
  - 요청과 같이 봇이 가진 역할보다 낮은 역할만 삭제할 수 있습니다.

> !역할 기본 테란 \
> !역할 기본 삭제 \
> !역할 부여 프로토스 \
> !역할 부여 @테스트#1234 \
> !역할 부여 삭제 \
> !역할 요청 추가 \
> !역할 요청 저그 \
> !역할 삭제 저그

## 제한

봇 명령어 사용을 채널/봇역할/명령어로 제한합니다.

?> 우선순위는 `명령어 역할 제한` → `봇 역할 제한` → `명령어 채널 제한` → `채널 제한` 순으로 제한됩니다.

- `!내서버`로 설정된 내용을 확인할 수 있습니다.
- 기본 명령어 또는 관리자 권한이 있으면 제한에 걸리지 않습니다.

### 채널로 제한

채널로 명령어 사용을 제한합니다.

- !제한 채널
  - 입력한 채널에서만 명령어를 사용할 수 있고, 최대 10개 채널까지 등록할 수 있습니다.
  - 등록한 채널에서 다시 입력하면 등록을 삭제합니다.
  - 관리자 권한이 있으면 제한에 걸리지 않습니다.
- !제한 채널 초기화
  - 등록된 채널 제한을 모두 삭제합니다.
  - 채널이 삭제되어 등록을 삭제할 수 없을 때 사용할 수 있습니다.

### 봇역할로 제한

봇역할로 명령어 사용을 제한합니다.

- !제한 봇역할
  - `서버 설정 > 역할`에서 봇에게 할당된 역할보다 순서가 높은 유저만 명령어를 사용할 수 있습니다.
  - 봇의 역할을 마우스로 드래그하여 순서를 변경할 수 있습니다.
  - 관리자 권한이 있으면 제한에 걸리지 않습니다.

### 명령어별 제한

특정 명령어를 특정 역할 또는 채널로 명령어 사용을 제한합니다.

- !제한 `명령어` `역할명`
  - 설정된 역할보다 높은(>=, 크거나 같은) 유저만 명령어를 사용할 수 있습니다.
  - 명령어를 끄고 싶다면 관리자 역할로 설정합니다. `!제한 계산 관리자`
  - 이 제한이 설정된 명령어는 `!제한 봇역할` 설정을 건너뜁니다.
- !제한 `명령어` 역할해제
  - 명령어에 제한된 역할을 삭제합니다.
- !제한 `명령어` 채널
  - 입력한 채널에서만 명령어를 사용할 수 있습니다.
  - 이 제한이 설정된 명령어는 `!제한 채널` 설정을 건너뜁니다.
- !제한 `명령어` 채널해제
  - 명령어에 제한된 채널을 삭제합니다.

> !제한 채널 \
> !제한 채널 초기화 \
> !제한 봇역할 \
> !제한 계산 테스트 \
> !제한 계산 역할해제 \
> !제한 계산 채널 \
> !제한 계산 채널해제

## 스팸

외부 서버 초대 링크를 차단합니다.

- !스팸 추방
  - 봇이 `멤버 추방` 권한을 가지고 있어야 합니다.
  - 현재 서버가 아닌 다른 서버의 초대 링크를 올리면 유저를 추방(내보내기=킥)합니다.
  - 서버에 다시 입장할 수 있고, 추방된 유저의 최근 메시지 기록을 삭제합니다.
- !스팸 차단
  - 봇이 `멤버 차단` 권한을 가지고 있어야 합니다.
  - 현재 서버가 아닌 다른 서버의 초대 링크를 올리면 유저를 차단(영구추방=밴)합니다.
  - 차단된 유저의 최근(1일) 메시지 기록을 삭제하고, 차단 해제 전까지 서버에 입장할 수 없습니다.
  - `서버 설정 > 차단`에서 차단한 유저를 확인할 수 있습니다.
- !스팸 해제
  - 설정된 스팸을 해제합니다.

> !스팸 추방 \
> !스팸 차단 \
> !스팸 해제

## 로그

유저 활동 로그를 메시지로 기록합니다.

- !로그 `입장|퇴장|추방|뮤트|스팸|해제:옵션`
  - 옵션 없이 `!로그`만 입력하면 입력한 채널에 모든 로그를 기록합니다.
  - 개별 로그가 설정되면 해당 채널에 기록합니다. 다시 입력하면 해제됩니다.
- !로그 입장 → 서버에 처음 입장한 유저를 표시합니다.
- !로그 퇴장 → 서버에서 사라진(추방/차단/나가기) 유저를 표시합니다.
- !로그 추방 → `!킥` 명령어로 추방된 내용을 기록합니다.
- !로그 뮤트 → `!뮤트` 명령어로 뮤트된 내용을 기록합니다.
- !로그 스팸 → `!스팸` 설정으로 처리된 내용을 기록합니다.
- !로그 해제 → 설정된 로그 기록을 모두 해제합니다.

> !로그 \
> !로그 입장 \
> !로그 추방 \
> !로그 해제

## 청소

입력한 개수 범위 내의 채팅 메시지를 삭제합니다.

?> `!제한 청소 역할명`으로 청소 명령어를 특정 역할만 제한해야 사용할 수 있습니다. \
사용자가 `메시지 관리` 권한이 있으면 제한에 걸리지 않습니다.

- 봇이 `메시지 관리` 권한이 있어야 합니다.
- 삭제한 메시지는 복구할 수 없으며, 2주 이상 지난 메시지는 삭제되지 않습니다.
- 채널의 전체 메시지를 삭제하려면 디스코드의 `채널 복제하기`로 채널을 재생성하고 기존 채널을 삭제하세요.
- !청소 `개수:숫자`
  - 명령어를 입력한 유저가 채널에서 `메시지 관리` 권한이 없으면 본인의 메시지만 삭제합니다.
  - 20개를 지정하면 최근 메시지 20개 중에 본인이 작성한 메시지만 삭제합니다.
- !청소 `개수:숫자` `@멘션|#태그`
  - 채널에서 `메시지 관리` 권한이 있을 때 개수 뒤에 @멘션 또는 #태그로 청소할 유저를 지정할 수 있습니다.
  - 20개를 지정하면 최근 메시지 20개 중에 @멘션 또는 #태그 유저가 작성한 메시지만 삭제합니다.
- !청소 `개수:숫자` `메시지ID`
  - 특정 메시지ID를 입력하여 해당 메시지 다음에 오는 메시지만 삭제합니다.
  - 메시지 삭제의 시작 지점을 지정하는 것과 같습니다.

> !청소 50 \
> !청소 25 @테스트 \
> !청소 75 테스트#1234 \
> !청소 10 123456789876543210

## 뮤트

채널에서 유저의 채팅을 금지합니다.

?> `!제한 뮤트 역할명`으로 뮤트 명령어를 특정 역할만 제한해야 사용할 수 있습니다. \
뮤트는 일시적인 제재방안으로 영구 뮤트는 디스코드의 서버 차단 기능을 사용하고 뮤트를 해제하세요.

- 봇이 `역할 관리` 권한을 가지고 있어야 합니다.
- 뮤트 역할이 설정된 채널에서 메시지 보내기(+연결)가 금지되고 서버 재입장 시에도 유지됩니다.
- !뮤트 역할 `역할명`
  - 뮤트 역할은 뮤트할 때 지정된 대상에게 부여하는 역할입니다.
  - 뮤트 역할을 설정하면 봇이 접근할 수 있는 채널에 뮤트 역할을 추가합니다.
  - 설정할 역할이 없어 새로 역할을 생성할 때 `@everyone` 역할 위에 `역할 권한 지우기`로 만든 아무 권한이 없는 역할을 권장합니다.
  - 새로운 채널이 생성되면 자동으로 뮤트 역할이 추가됩니다.
- !뮤트 역할 삭제
  - 설정한 뮤트 역할을 삭제하고 채널에 추가한 뮤트 역할을 제거합니다.
  - 특정 채널에서 뮤트를 제외하려면 채널 편집에서 뮤트 역할을 직접 삭제하세요.
- !뮤트 `@멘션` `사유:옵션`
  - 뮤트하려는 대상을 @로 호출합니다. 한 번에 한 명만 가능합니다.
  - 봇 또는 자신의 역할보다 높은 대상은 뮤트할 수 없습니다.
  - 대상이나 채널의 역할 설정에 따라 뮤트되지 않을 수 있습니다.
  - 사유를 입력하면 로그 내용에 포함됩니다.
- !뮤트 해제 `@멘션`
  - 뮤트 해제하려는 대상을 @로 호출합니다.
  - 뮤트할 때와 조건이 같습니다.
- !뮤트 목록 `페이지:숫자`
  - 뮤트한 대상 목록을 보여줍니다.
  - 서버에서 나간 유저는 ✖로 표시되고 아이디는 뮤트 해제할 때 사용됩니다.
  - 영구 뮤트는 서버 차단 후에 목록을 보고 뮤트를 해제하세요.
- !뮤트 삭제 `아이디`
  - 뮤트한 대상이 서버를 나가 멘션할 수 없을 때 아이디로 뮤트를 삭제할 수 있습니다.
  - 뮤트 목록에서 보여주는 아이디를 입력하여 삭제합니다.
- !뮤트 초기화 `@멘션`
  - 뮤트된 대상을 모두 삭제합니다.
  - @멘션에는 자기 자신을 호출해야 합니다, 실수 입력 방지 차원
- `!로그 뮤트`로 명령어 사용내역을 기록할 수 있습니다.

> !뮤트 역할 Mute \
> !뮤트 역할 삭제 \
> !뮤트 @테스트#1234 \
> !뮤트 해제 @테스트#1234 \
> !뮤트 목록 \
> !뮤트 삭제 123456789098765432 \
> !뮤트 초기화 @테스트#1234

## 슬로우
